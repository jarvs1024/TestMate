## P0 范围

按设计方案 §12,P0 目标:Monorepo 脚手架 + 1 个端到端 demo(log 诊断 SSE 流式),2 周。

## 已完成

### T1 Monorepo 目录结构
- `backend_gateway/` (FastAPI)
- `frontend_web/` (Vue 3)
- `ai_engine/` (Dify + RAGFlow 配置,占位)
- `deploy/` (docker-compose)
- `scripts/` (dev.sh / stop.sh)
- `docs/` (specs / api)

### T2-T4 后端骨架
- FastAPI app + Pydantic config + SQLAlchemy 2.0 async
- 6 个 API 路由:`/health` `/auth` `/machines` `/kb` `/diagnose` `/notify`
- JWT 鉴权 + RBAC(`require_role` 依赖)
- Dify SSE 流式客户端
- 2 张表:`users` / `machines`(P0 简化,启动时 create_all)
- log 大文件截断预处理(50KB 头 + 50KB 尾)

### T5-T7 前端骨架
- Vue 3 + Vite + TS + Tailwind + Element Plus
- 三轴 MainLayout(左导航 / 中工作区 / 右 RAG 抽屉)
- Pinia user store(JWT + 角色)
- 4 个页面:Login / LogDiagnosis(demo)/ KnowledgeBase(占位)/ Settings(占位)
- `/api` 走 Vite dev proxy,生产走 Nginx 反代

### T8 部署
- `deploy/docker-compose.yml`:MySQL + Redis + Backend + Frontend 一键起
- Dify / RAGFlow 在 P0 暂不接入(占位,生产单独 compose 启)
- `scripts/dev.sh`:Mac 本地起(分开起后端 / 前端,不依赖 docker)
- `scripts/stop.sh`:关掉本地进程

## 端到端 demo 流

```
[用户] 浏览器打开 http://localhost:8080
  → 登录(admin / admin123,P0 手动建用户)
  → 进入 /diagnosis
  → 点「填入示例 log」(或自己贴)
  → 点「开始诊断」
    → 前端 POST /api/v1/diagnose/stream (JWT 鉴权)
    → 中台截断 log → 转发 Dify /chat-messages (SSE)
    → 中台边收边 yield,前端边读边渲染
  → 右侧 Monaco 风黑底色面板流式显示 AI 结论
```
